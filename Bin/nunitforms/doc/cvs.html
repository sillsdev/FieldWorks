Contributing

  1. Post the code to the developer mailing list or simply email it to me.

  2. Write unit tests for any new features that you are adding.

  3. Post a message to the list (or in the email you send me) assigning copyright of the code to Luke Maxon. (I will of course acknowledge your contribution any way you like but need to be able to release it without worries.)

  4. It would be best if your patch is done against the most recent version of the code in CVS because it is changing quickly.

Thoughts? Ideas? How would you prefer to contribute?
