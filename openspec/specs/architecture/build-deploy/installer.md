# Installer

## Purpose

Describe installer build and packaging patterns for FieldWorks.

## Context

FieldWorks uses WiX 3 (default) and WiX 6 (opt-in) installers. The installer build is invoked by `build.ps1` via `InstallerBuild.proj /t:BuildInstaller` after the main traversal build completes.

## Installer Pipeline

### Entry Point

`build.ps1 -BuildInstaller` passes `-InstallerToolset Wix3` (default) or `-InstallerToolset Wix6` to control which pipeline runs.

### WiX 3 (Default)

- Entry: `Build/Installer.Wix3.targets` → imports `Build/Installer.legacy.targets`
- Harvests traversal build outputs into WiX fragments
- Produces MSI package and burn bundle
- Located in `FLExInstaller/`

### WiX 6 (Opt-in)

- Entry: `Build/Installer.targets`
- SDK-style WiX project
- Located in `FLExInstaller/wix6/`

### Build Flow

```
build.ps1 -BuildInstaller
└── msbuild InstallerBuild.proj /t:BuildInstaller
    ├── InstallerToolset=Wix3 (default)
    │   └── Installer.Wix3.targets → Installer.legacy.targets
    │       └── Harvest → Compile → Link → Bundle
    └── InstallerToolset=Wix6
        └── Installer.targets
            └── SDK-style WiX build
```

## Constraints

- Installer builds require a completed traversal build — never run standalone.
- Registration-free COM: do not register COM globally; rely on reg-free manifests generated by `RegFree.targets`.
- Keep installer builds in sync with traversal build outputs.

## Anti-patterns

- Running installer builds without a current traversal build.
- Registering COM components globally instead of using reg-free manifests.
