AdaptIt Target Word Guesser

This transducer can be used in an AdaptIt adaptation project to guess target words based on the Source/Target word pairs in the AdaptIt Knowledge Base.

Forthcoming versions of AdaptIt will have this functionality built-in, but until then (roughly Fall ’07), this functionality is available thru this Target Word Guesser for AdaptIt EncConverter plug-in.

 

This transduction engine is not installed by default. If you want to use it, you must explicitly request the AdaptIt entry in the Transduction Engines section of the SILConverters 2.6 (and newer) installer.

Setup Tab

Here is the setup tab for the Target Word Guesser for AdaptIt:

Figure 1: Target Word Guesser for AdaptIt Setup [1], [2], [3], [4]

 

1.      Choose which version of AdaptIt (i.e. Non-Roman/Unicode vs. Legacy/Ansi) that you use to create the (XML) Knowledge Base.

2.      Choose the project desired in the Projects list box. This is automatically populated from the projects available on the local machine for the current user. You will want to choose the same project for which the guesser is to be used (see below).

3.      If you want the converter to be permanently added to the System Repository, then you must click the Save in System Repository button (see below).

4.      You can click on the Test Area tab in order to test the converter with some sample data.

For example, given an AdaptIt knowledge base with the following word pairs:

Source

Target

boyes

boyez

girles

girlez

guyes

guyez

 

Then for the source word pearles, the guesser will return pearlez.

Saving the converter in the System Repository:

By default, if you click the Apply or OK buttons, the configured converter will be returned to the client application as a temporary converter for temporary use. Once the client application (e.g. AdaptIt) is closed or releases the converter, it will no longer be available. If you want the converter to be permanently available to client applications, then you must explicitly add it to the System Repository using the Save in System Repository button.

When you click this button, the following dialog box will be displayed to query for a friendly name by which the converter will be known in client applications:

 

For the AdaptIt Target Word Guesser, the default name will be the AdaptIt project name preceded by the string Target Word Guesser for. However, you can change it to something more meaningful if desired.

The Advanced... button can be used to enter further, optional information about this converter which is also put into the System Repository and can be used by various client applications. Click on this button to bring up the following dialog box:

 

Though these values are not necessary for the operation of the converter, they can be helpful to various client applications. For example, the Clipboard EncConverter can be configured to filter the list of displayed converters based on the Encoding Name and/or the Transduction Type configured here.

Configuring AdaptIt to use the Guesser EncConverter

The Knowledge Base accessed by this converter type is created by AdaptIt v. 3.1 or newer.

Open the AdaptIt project you want to use the guesser algorithm with and ensure the following:

·        When you open a document to process, make sure the Save Document and KB as XML box is checked (as it is by default).

·        On the Tools menu, select the SIL Converters... menu item:

Figure 2: AdaptIt Tools Menu

 

Note that these menu items will be grayed out (as here) unless you have EncConverters v 2.2 or newer installed on your system. This utility is available as part of FieldWorks 3.1 or it can be downloaded from this website: http://scripts.sil.org/EncCnvtrs

 

Depending on the version of AdaptIt you are using, you may need to copy the file SilEncConverters22.tlb from the folder,

C:\Program Files\Common Files\SIL\2.6.0.0

into

C:\Program Files\Common Files\SIL

The presence of the type library file in that folder indicates to AdaptIt that SILConverters is installed.

Selecting this menu item will bring up the Choose SIL Converter dialog box which shows you the EncConverter that will be used to pre-process the Source word before putting it in the Target word box (though, initially this dialog box will be empty):

Figure 3: Choose SIL Converter dialog box

 

·        Click the ... browse button

The Choose an Existing Converter dialog box will appear.

·        Click the Add New button.

The Choose a Transduction Engine dialog box will appear.

·        Select the Target Word Guesser for AdaptIt item and click the Add button.

The Target Word Guesser for AdaptIt Setup dialog box will appear.

At this point, you should follow the instructions above for configuring the Guesser EncConverter. Choose the same AdaptIt project that you are configuring this converter for.

In AdaptIt when you’ve typed a target word and hit the enter key to accept a Source/Target word association, this information is not automatically saved in the knowledge base until you click the Save button (or type Ctrl+S). Until you save the knowledge base, the words will not be available to the guesser algorithm. To insure that the word pairs are available to the guesser algorithm, you should save your progress often.

Once the guesser algorithm has been added directly to AdaptIt (rather than via this EncConverter transducer), this won’t be necessary. But until then, you must save often for this approach to work.

·        Click OK as needed to return to AdaptIt.

The Guesser transducer is now ready to provide guesses for the target words based on the Source/Target word associations you’ve previously added.
