ICU Transliterators

The International Components for Unicode (ICU) provides a wide variety of transliteration services for Unicode data. With this EncConverters' wrapper on ICU Transliterators, you can configure one of the built-in transliterators (e.g. "Devanagri-Latin", which follows the ISCII standard defined romanization scheme for Indic data), or write custom transform rules to process text data.

See http://icu.sourceforge.net/userguide/Transform.html for more details on the use and syntax of ICU Transliterators.

Setup Tab

Here is the setup tab for the ICU Transliterators EncConverter:

1.      If you want to use one of the built-in transliterators, click the Built-in transliterator radio button at the top and then select the transliterator you want to use from the list box shown above.

2.      If you want to write a custom transliterator using the syntax described on the webpage referred to above, then click the Custom transliterator radio button and enter the transliterator syntax in this box.

3.      If the Custom transliterator radio button is selected, then the Previous Custom Rules combo box will be enabled showing a few examples of useful custom rules as well as any others that you've written. The Delete button can be used to remove unwanted rules.

4.      If you want the converter to be permanently added to the System Repository, then you must click the Save in System Repository button (see below).

5.      You can click on the Test Area tab in order to test the converter with some sample data.

6.      You can add Advanced information (encoding names, process types, etc) using the Advanced tab.

Saving the converter in the System Repository:

By default, if you click the Apply or OK buttons, the configured converter will be returned to the client application as a temporary converter for temporary use. Once the client application (e.g. FieldWorks or Word) is closed or releases the converter, it will no longer be available. If you want the converter to be permanently available to client applications, then you must explicitly add it to the System Repository using the Save in System Repository button.

When you click this button, the following dialog box will be displayed to query for a friendly name by which the converter will be known in client applications:

 

For the ICU Transliterators, the default name will be the ICU display name for the transliterator or the complex rule itself. However, you can change it to something more descriptive if desired (e.g. "Transliterator to Latin").
